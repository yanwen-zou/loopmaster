Abstract

自进化智能体正快速逼近「让机器人自己学会新技能」的目标，但绝大多数系统的证据来自仿真基准， 并在自由生成代码 / 无约束技能库上自我改写——这带来三类被学术界持续关注的隐患： 工具幻觉（planner 臆造不存在的能力）、仿真证据泄漏 / 成功作弊（用仿真里的 check_task_success 冒充真机成功）， 以及技能库漂移（self-evolving skill library 静默腐化）。

我们提出 LoopMaster：一个面向真实双臂移动机器人（开源本体 Hei-Rebot-Lift）的自进化智能体框架。 它把一次运行拆成四个文件化契约的角色——Handler / Strategist / Worker / Auditor——并引入两条核心机制： (1) 注册表锚定规划，Strategist 只能选用 SKILL.md 里声明过的技能，从结构上杜绝工具幻觉； (2) 对抗式闭环审计，Worker 在每个控制动作后强制注入 observe 采证，独立 Auditor 绝不凭指令回执判成功、 并主动检测「仿真泄漏」——证据来自仿真即判 blocked。 当发现缺失能力（research_needed）时，Auditor 以轨迹为证据提案新技能， 经语法/导入/签名门控校验+失败回滚后热重载注册表并重跑——实现可审计、可回滚的安全自进化。 一名普通「小老板」即可用人类可读的 markdown+python 教会它新技能。

我们在学术上解决了什么

把「自进化 Agent」从仿真榜单带到真机货架，本质要回答的是一组可信性问题：

P1 · GROUNDING

工具/技能幻觉

LLM planner 常臆造不存在的 API/技能。→ 注册表锚定规划：计划只能引用已声明技能，执行前做集合校验。

P2 · EVIDENCE

仿真泄漏 / 成功作弊

自进化系统易用仿真成功冒充真机成功。→ observe-after-control 强制采证 + Auditor 仿真泄漏检测，证据来自 sim 即 blocked。

P3 · LIFECYCLE

技能库漂移与不安全自改

skill library 会静默腐化、自改写可能损坏既有能力。→ 门控自创作：语法/导入/签名校验 + 失败自动回滚 + 独立审计。

P4 · TRUST

Agent 回路不可复现/不可审计

单体 LLM 回路是黑盒。→ 四角色文件化契约（plan/trace/review）+ LoopViz 全程可视化，任一步可回放。

P5 · SAFETY

自进化中的真机安全

自主改写时的真机风险。→ 安全内建：控制预检可拒绝、自动 stop_motion、末端 z 与底盘速度/时序硬边界。

P6 · TEACHABILITY

教新能力的成本

加新技能通常需专家。→ 技能=人类可读 markdown+python，小老板可教、可批准；也可由 agent 自证自写。

Method · 四角色闭环 + 技能注册表

① 四角色文件化契约（reproducible role graph）

一次运行被拆成四个各有明确契约、以文件交接的子代理：Handler 掌管运行与机器人连接；Strategist 产出 plan.md；Worker 执行并写 trace.jsonl；Auditor 复核并写 review.md，判定 done / retry / blocked / research_needed。全程产物落盘，天然可复现、可审计（即 LoopViz 所可视化的对象）。

② 注册表锚定规划（registry-grounded planning）

每个技能是一个带 SKILL.md 声明（name / args schema / 描述）+ policy.py（dispatch()）的一等对象。 Strategist 只在注册表的技能集合内选择与编排，执行前用 available = {s.name for s in skills.list()} 校验—— 从结构上消除「自由生成代码」范式里的工具幻觉与仿真-only 谓词漂移。

③ 对抗式闭环审计（adversarial, evidence-first audit）

Worker 在每个控制动作后自动注入 observe 采集前后状态；Auditor 独立复核，契约明确 「never mark done from action_sent alone」，要求看到状态确实按预期变化，并用采样趋势/区间/稳定窗口而非单点相等来判断。 同时检测 {robotwin, sim, simulation, check_task_success} 等仿真泄漏词——证据来自仿真即判 blocked。

④ 门控自创作（gated self-authorship：「create_skill 本身是一个 skill」）

当 Auditor 判 research_needed，它以轨迹为证据提出完整的 SKILL.md+policy.py 提案； 系统 py_compile 语法检查 → 动态导入 → 校验 dispatch(context,args) 签名 → 任一失败回滚到备份； 通过后热重载注册表并重跑闭环。新技能自此成为「已注册、被审计」的一等能力。这条回路让系统在同一次会话内发现缺陷→自写修复→复用。

⑤ 安全内建 & 感知/操作栈

Worker 预检可对不安全动作返回 proceed=false；失败后自动 stop_motion；末端 z≥0.06m、底盘速度/时序（duration_s / settle_s / velocity_limit）等硬边界写进技能契约。

Key Contributions

1
注册表锚定规划：以声明式技能契约约束 LLM 规划，从结构上消除工具幻觉与仿真-only 谓词漂移。
2
对抗式闭环审计 + 仿真泄漏检测：observe-after-control 强制采证，独立 Auditor 只认真机证据，直击自进化里的「成功作弊 / sim leak」。
3
门控、可回滚的自创作技能：Auditor 提案 → 校验 → 回滚保护 → 热重载重跑，缓解 self-evolving skill library 的「library drift」与不安全自改。
4
文件化四角色契约 + LoopViz 可视化：把 agent 回路做成可复现、可回放、可审计的系统，而非黑盒单体循环。
5
真机优先 + 小老板可教：全流程跑在开源双臂移动本体 Hei-Rebot-Lift 上；技能是人类可读的 markdown+python，非专家亦可教学与批准。

定位小结：ASPIRE 用进化式程序搜索在仿真里最大化能力覆盖；Maestro 用 VLM 编排预训练 VLA 模块做零样本泛化； 自进化技能库范式（Voyager/ExpeL 系）以「成功即入库」增长能力。LoopMaster 的差异化在于“可信的真机自进化”—— 用注册表锚定 + 对抗式闭环审计 + 门控自创作，把安全、证据与可审计性放在能力增长之前。

Real-Robot：48 小时里发生了什么

  • 顿悟：「create_skill 本身应该是一个 skill」——自进化回路由此闭合。
  • 机器人动了，而且没写一行 code：由 agent 自主规划-执行-审计驱动。
  • 撞桌子后，人类只加了一条硬边界 z > -0.15；agent 随后自己拍照、自己写了 navigation skill。
  • 为让它意识到「时间」，agent 获得 meta/timer skill；随后学会 shortcut：直接调用 cache 轨迹抓取。
